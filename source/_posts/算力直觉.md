## 模型的参数量和存储大小的关系

核心计算 ： 模型存储 = 参数量 $\times$ 每个参数的字节数

| 模型精度 | 每个参数占用 |
| -------- | ------------ |
| FP32     | 4 Bytes      |
| FP16     | 2 Bytes      |
| INT 8    | 1 Bytes      |
| INT 4    | 0.5Bytes     |

例如

FP32下，4B参数 => 4 * 10 ^ 9（4 Billion） 个参数 => 占用存储 8GB

## 显存占用 !== 模型大小

$$
显存=模型权重 + KV \ Cache +activation
$$



**经验公式**

```tex
推理显存=1.2 ~ 1.5 倍模型大小
```

## 训练显存（比推理大很多）

训练需要保存：

```
参数
梯度
优化器状态
activation
```

经验公式：

```
训练显存 ≈ 6x ~ 8x 模型大小
```

例如

| 模型 | FP16大小 | 训练显存 |
| ---- | -------- | -------- |
| 7B   | 14GB     | 100GB    |
| 13B  | 26GB     | 200GB    |
| 70B  | 140GB    | 1TB      |

所以训练通常需要：

```
多卡 + ZeRO / FSDP
```

## FLOPs相关

训练 Transformer LLM

```
训练 FLOPs = 6 * 参数量 * token数
最优 token 数 = 20 * 参数量
=>
训练FLOPs = 120 * 参数量 ^ 2
```
