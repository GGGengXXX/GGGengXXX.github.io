Thinking 0.2

add the file 和 stage the file 都可以用命令 git add <file>

commit 可以用 命令 git commit -m "xxx"

Thinking 0.3

  1. 如果 print.c 在暂存区中，可以使用 git checkout -- <file> 命令，将暂存区的内容覆盖到工作区
  2. 前提是之前已经有过提交，先使用 git reset HEAD <file> 撤销暂存区的删除操作，再使用 git checkout -- <file> 将暂存区中的内容覆盖到工作区
  3. 使用 git rm --cached <file> 将文件从暂存区中移出

Thinking 0.5

  • echo first
    • 在控制台中输出 first
  • echo second > output.txt
    • 将 second 输出到文件 output.txt 中
  • echo third > output.txt
    • 将 third 输出到文件 output.txt 中
    • 原先 output.txt 中的内容会清空
  • echo forth >> output.txt
    • 将 forth 追加输出到 output.txt 中
    • 原先 output.txt 中的内容保留，新的内容接在原内容的末尾

Thinking 0.6

./command 中的内容如下

  • 只需要在原 文件内容 前面加上 echo xxx > test 注意 $ 这个符号需要加上 \ 进行转义

  • 执行 ./command 后，test 文件中内容如下

    • echo Shell Start...
echo set a = 1
a=1
echo set b = 2
b=2
echo set c = a+b
c=$[$a+$b]
echo c = $c
echo save c to ./file1
echo $c>file1
echo save b to ./file2
echo $b>file2
echo save a to ./file3
echo $a>file3
echo save file1 file2 file3 to file4
cat file1>file4
cat file2>>file4
cat file3>>file4
echo save file4 to ./result
cat file4>>result

echo echo Shell Start 和 echo 'echo Shell Start' 没有区别，都是输出echo Shell Start

echo echo $c>file1 与 echo 'echo $c>file1' 不同，前者会将 echo $c 输出到 file1 中，而后者会将 echo$c>file1 输出到控制台中
